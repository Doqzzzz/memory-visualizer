# Memory Visualization System — Design Spec

## 概述

基于纯前端（HTML/CSS/JS）的 Web 内存可视化系统，用于教学演示变量赋值、列表存储、引用、浅拷贝和深拷贝的内存变化过程。

- 目标用户：大学生提交作业、课堂演示
- 技术栈：纯 HTML + CSS + JS，零依赖
- 交互模式：预设场景 + 可编辑代码 + 逐步动画

## 架构

```
输入层 → 解释器 → 状态管理 → 渲染层
(代码编辑器)  (解析执行)  (事件/快照)  (Canvas/SVG)
```

**4 层单向数据流：**

1. **输入层** — 代码编辑器（textarea）+ 预设场景选择器，输出代码字符串
2. **解释器** — 解析代码行，模拟内存操作，输出 `MemoryEvent[]` 事件序列
3. **状态管理** — 维护当前步骤号，累积事件到当前步的快照，提供前进/后退
4. **渲染层** — Canvas 绘制盒子 + 箭头，SVG 备选

## 页面布局

三区布局（左侧预设列表 + 中间代码编辑 + 右侧内存视图）：

- 左侧 ~180px：预设列表，高亮当前选中，一键切换
- 中间 ~35%：代码编辑区 + 运行/重置按钮
- 右侧 ~50%：Canvas 内存视图 + 底部步骤控制条

## 解释器

### 数据结构

**MemoryObject** — 内存中的一个实体：
```
{ address, type: "int"|"float"|"str"|"list"|"pointer",
  value: 基本类型的值,
  refs: [address, ...]  // 列表类型专用，存储元素引用地址
}
```

**MemoryEvent** — 一条操作记录：
```
{ step, action: "create"|"bind"|"modify"|"destroy"|"set_element",
  varName, address, snapshot: { ... }  // 该步之后的完整内存快照
}
```

### 支持的 Python 操作

| 代码 | 行为 |
|------|------|
| `a = 1` | 创建 int 对象，变量绑定 |
| `b = a` | b 指向 a 的同一对象（引用） |
| `b = 2` | b 重新绑定，a 不变 |
| `lst = [1, 2]` | 创建 list 对象，refs 指向各元素 |
| `lst[0] = 99` | 修改列表元素引用 |
| `lst2 = lst.copy()` | 浅拷贝：新 list 对象，refs 相同 |
| `lst2 = deepcopy(lst)` | 深拷贝：递归复制所有对象 |
| `matrix = [[1,2],[3,4]]` | 2D 列表：外层 refs 指向内层 list |

### 支持的 C 操作

| 代码 | 行为 |
|------|------|
| `int a = 1` | 栈分配，直接存值 |
| `int *p = &a` | 指针存地址 |
| `*p = 2` | 通过指针修改原值 |

## 渲染

- **盒子**：Canvas `fillRect` + `strokeRect`，顶层变量盒子 + 列表子元素盒子
- **箭头**：Canvas `bezierCurveTo`，曲线避免重叠
- **布局**：从左到右横向排列，列表元素在父盒子下方展开
- **动画**：新建=绿色闪烁 → 修改=黄色闪烁 → 销毁=红色淡出

### 渲染流程

```
状态管理器 → 计算布局坐标 → 绘制盒子 → 绘制箭头 → 应用高亮动画
```

## 预设场景（6 个）

1. **Python 基本赋值** — `a=10; b=a; b=20`，演示不可变对象重绑定
2. **C 基本类型 + 指针** — `int a=1; int *p=&a; *p=2`，演示指针修改原值
3. **Python 列表** — `lst=[1,2,3]; lst[0]=99`，演示列表元素引用变化
4. **2D 列表** — `m=[[1,2],[3,4]]; m[0][1]=99`，演示嵌套引用结构
5. **浅拷贝** — `a=[1,2]; b=a.copy(); b[0]=99`，演示浅拷贝元素引用共享
6. **嵌套列表浅拷 vs 深拷** — `a=[[1,2],[3,4]]; b=a.copy(); c=deepcopy(a); b[0][0]=99`，核心对比

所有场景代码可编辑，修改后可重新运行。

## 约束与边界

- 不实现完整 Python/C 解释器，仅覆盖作业所需语法子集
- 不依赖任何第三方库或框架
- 不支持函数调用、循环、条件语句
- 模拟地址使用短十六进制（如 `0x0001`），非真实内存地址
- 每步执行一行代码，不支持多行语句
